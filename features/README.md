# Features

This directory contains documentation for planned and in-progress features. Each feature has its own Markdown file with:

- **Overview** — What the feature is and why it matters
- **Goal** — Specific, measurable outcome
- **Assumptions** — What we're taking for granted
- **Plan** — Implementation steps with `- [ ]` checkboxes to track progress

## Adding a New Feature

1. Create a new file like `feature-name.md`
2. Use the template structure with Overview, Goal, Assumptions, and Plan sections
3. Add checklist items to Plan as you break down the work
4. Check off boxes as you complete each step
